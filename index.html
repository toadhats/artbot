Artbot 420

Aleatoric drawing in Processing.

A bunch of functions thrown haphazardly together with almost no user interaction beyond editing the code. However, you can:

  • Click anywhere to pause the drawing
  • Press Shift-S to save the drawing to /img
  • Press Backspace to clear the canvas

Functions

Drip()

Distributes shapes (circles) evenly around a centre point using a Gaussian distribution. Looks like paint drips, or something. When used alongside squiggle() it adds a sense of depth in the centre, and soft detail noise over the rest, or however you want to think of it.

Squiggle()

Kinda 'scribbles' randomly, as if with a permanent marker. Looks cool with evolving colour. The offset allows multiple "squigglers" to be instantiated without giving each its own t variable for the perlin noise function. Without different offsets, they'd follow the same path, which is generally not what I want. Each call draws a single point along the path, so call in a loop.

Signal()

Draws a squiggly line across the page. Like a squiggler, but the x moves from left to right, while the y value is randomized over time via perlin noise. Draws one whole line per call, unlike the squiggler.

Uses its x-coord as a t value, which means we're basically graphing the perlin function.Offset determines what 'part' of the perlin space we're graphing. Scale controls the speed through perlin space, ie 'jaggedness'.
